# Database Changes Log

This file tracks all database schema changes, migrations, and data modifications to ensure the team has a single source of truth for the database state, following the mandated user rule.

## Format
- **Date:** YYYY-MM-DD
- **Change summary:** What changed
- **Reason:** Why this change was needed
- **Migration file name:** Make sure the corresponding DB migration file exists in `backend/migrations/`
